📘 Project Overview

This project aims to build a simple file copy tool called mycp using C language and Linux system calls. It helps students understand how files are handled at the system level.

This project involves implementing a simplified version of the cp command in C using system calls like open, read, write, and close. It also includes Bash scripts to test the implementation and analyze system calls and memory management using valgrind.

🛠️ Tasks Summary

  • 1- Complete Makefile with build/test/clean rules
  • 2- Implement file operations in file_ops.c
  • 3- Define copy settings and logic in mycp.c
  • 4- Parse arguments and validate inputs in main.c
  • 5- Write test_copy.sh to verify file copying
  • 6- Use strace in system_call_analysis.sh to compare with cp
  • 7- Use valgrind to catch memory leaks
  • 8- Document team effort and design in README.md

🧪 Analysis & Testing

Valgrind: Run with:

valgrind --leak-check=full ./mycp -f file1.txt file2.txt

System Calls: Compare mycp and cp with:

./system_call_analysis.sh

Results are saved in mycp_trace.txt and cp_trace.txt.
